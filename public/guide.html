입찰 검토 보조 시스템

HVAC 해저·지중 케이블 입찰 검토 보조 시스템 · 사용설명서 v1.0 · 2026-05

이 시스템은 입찰 사양서(Tender Spec) PDF를 업로드하면 AI가 기술 요구사항을 자동 추출하고, 실무자·팀장·부문장 3단계 검토 워크플로우를 지원하는 보조 도구입니다.

⚠ 중요
AI가 생성한 모든 결과는 초안(Draft)입니다. 반드시 담당자가 내용을 확인하고 판정하세요. 시스템은 보조 도구이며, 최종 판단은 사람이 합니다.

역할과 권한

역할 주요 권한 접근 범위
실무자 입찰 등록, PDF 업로드, AI 분석, Comply/Deviation 마킹, 검토 요청 제출 본인이 등록한 입찰만
팀장 제출된 분석 검토, Comply 수정, 승인/반려 모든 입찰 열람, 제출된 것만 수정
부문장 팀장 승인된 분석 최종 검토, 검토의견 초안 생성, 최종 승인/반려 모든 입찰 열람

PDF 업로드 & AI 분석

상세 페이지 하단 [문서] 영역에서 PDF 파일 업로드 Tender Spec PDF 파일을 선택하거나 드래그합니다.
[AI 분석 시작] 버튼 클릭 Claude AI가 PDF 전문을 읽고 기술 요구사항을 자동 추출합니다. 수십 초 소요될 수 있습니다.
추출 결과 확인 분석이 완료되면 시스템특성, 기술요구사항 목록이 표시됩니다.
(선택) 시스템특성 수정 전압, BIL/SIL, 단락용량 등 AI가 추출한 값을 확인하고 필요 시 직접 수정합니다.
(선택) 표준 규격 매칭 [표준 매칭] 버튼을 클릭하면 IEC/IEEE/CIGRE 관련 표준이 자동으로 연결됩니다.
⚠ 주의
PDF 텍스트 추출이 잘 안 되는 경우(스캔본, 보호 PDF 등)에는 일부 항목이 누락될 수 있습니다. 추출 결과 상단에 "[내용 잘림]" 메시지가 표시될 경우 원본 PDF와 대조하세요.

Comply 마킹

각 요구사항에 대해 자사 능력 기준 적합성을 판정합니다.

Comply 마킹 버튼
✓ 부합 ✗ 불부합 검토중
판정 의미 추가 입력
부합 자사 표준·능력으로 대응 가능 없음
불부합 자사 능력 초과 또는 대응 불가 불부합 사유 (필수)
검토중 추가 검토 필요, 미확정 없음
미판정 아직 판정하지 않은 상태 없음
💡 팁
같은 버튼을 다시 누르면 마킹이 해제됩니다. 불부합 사유는 내보내기(Excel/Markdown)에 포함됩니다.

Deviation 마킹

발주처 요구사항에서 이탈·불명확·가정이 있는 항목을 표시합니다.

Deviation 마킹 버튼
Deviation Clarification Assumption
유형 사용 기준
Deviation 자사가 사양서 요구와 다른 방식으로 이행하는 경우. 발주처 공식 서신 필요.
Clarification 요구사항이 불명확하거나 모호하여 발주처 확인이 필요한 경우.
Assumption 발주처 확인 전 자사가 임시로 전제한 사항.

사용 방법

해당 요구사항 행에서 유형 버튼 클릭 Deviation / Clarification / Assumption 중 선택
메모 입력 (선택) 이탈 근거, 질의 내용, 가정 내용을 간략히 입력합니다.
같은 버튼 재클릭 시 마킹 해제

코멘트 작성

분석 단위로 코멘트를 남길 수 있습니다. 팀원 간 소통에 활용합니다.

상세 페이지 하단 [코멘트] 영역에서 내용 입력
[등록] 버튼 클릭
기존 코멘트에 답글 달기 코멘트 하단 [답글] 버튼으로 1단계 답글을 달 수 있습니다. (최대 2단계)

검토 요청 제출

마킹 완료 후 상세 페이지 상단 [검토 요청] 버튼 클릭 상태가 작성 중 → 팀장 승인 대기로 변경됩니다.
제출 후에는 마킹 수정 불가 팀장이 반려하면 DRAFT로 복귀되어 다시 수정 가능합니다.
⚠ 주의
검토 요청 후에는 본인도 내용을 수정할 수 없습니다. 모든 마킹이 완료된 후 제출하세요.

요구사항 검토

입찰 상세 페이지 상단 요약 카드 확인 전체 요구사항 건수, 부합/불부합/검토중/미판정/RISK 건수를 한눈에 파악합니다.

코멘트 확인 및 답글 실무자 코멘트를 확인하고 필요 시 답글을 남깁니다.

초안 내용 검토 및 수정 생성된 초안을 직접 편집합니다. AI 초안은 참고용이며 반드시 검토 후 사용하세요.

FAQ

Q. 검토 요청 후 내용을 수정하고 싶어요.

팀장에게 반려를 요청하세요. 반려되면 상태가 DRAFT로 복귀되어 수정 가능합니다.

Q. PDF 분석 결과가 이상하거나 항목이 빠져있어요.

AI가 추출한 결과는 원본 PDF 품질에 따라 달라집니다. 스캔 품질이 낮거나 암호화된 PDF의 경우 추출이 불완전할 수 있습니다. 항목을 직접 수정하거나 추가할 수 있습니다.

Q. 같은 입찰을 다시 분석하고 싶어요.

상세 페이지에서 [재분석] 버튼을 클릭합니다. 기존 분석 결과가 새 결과로 교체됩니다. DRAFT 상태에서만 가능합니다.

Q. 표준 규격이 잘못 연결됐어요.

현재 버전에서는 개별 표준 연결 수정 기능이 없습니다. [표준 매칭] 버튼을 다시 눌러 재매칭하거나 팀 관리자에게 문의하세요.

Q. Deviation과 불부합의 차이가 무엇인가요?

구분 의미
불부합 자사 능력·표준으로 요구사항을 충족하지 못하는 경우 (기술적 한계)
Deviation 자사가 요구사항과 다른 방식으로 이행하겠다고 발주처에 공식 통보하는 경우

한 항목에 두 마킹이 모두 있을 수 있습니다.

Q. 검토 이력은 어디서 볼 수 있나요?

상세 페이지 하단 [검토 이력] 영역에서 모든 액션(제출, 승인, 반려)과 사유, 타임스탬프를 확인할 수 있습니다.

Q. 계정 추가나 비밀번호 변경은 어떻게 하나요?

현재 PoC 버전에서는 시스템 관리자가 직접 처리합니다. 관리자에게 문의하세요.
