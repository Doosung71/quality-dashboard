## 고도화 예정 항목 (Claude가 수립한 계획)

### Step 1 — UX 핵심 개선
1. 탭 전환 시 첫 번째 시험장 자동 선택
2. 노후 설비 행에 연한 빨간 배경 강조
3. 간트 차트: 월별 세로 그리드라인 추가 + 오늘 기준선 강화 (빨간 실선 + 날짜 라벨)

### Step 2 — 시각화 강화
4. KPI 카드에 노후 비율 프로그레스 바 추가
5. 시험장 리스트 행에 설비 상태 미니 도트 (신규·정상·노후 색상 표시)
6. 설비 테이블 핵심 컬럼 재정렬 + 덜 중요한 컬럼 접기/툴팁

### Step 3 — 정보 구조 개선
7. 시험장 선택 없이도 사이트 전체 설비를 볼 수 있는 "전체 보기" 모드
8. 간트 체크박스에 툴팁으로 "간트 필터" 용도 안내

## 확인할 질문

### Q1. 대시보드 UX 모범 사례 — "자동 선택" 패턴
- 리스트 패널에서 첫 항목 자동 선택(master-detail 패턴)이 대시보드 UX에서 권장되는 사례인가?
- 언제 자동 선택이 도움이 되고, 언제 사용자 혼란을 주는가?
- 참고할 수 있는 공식 UX 가이드라인이나 디자인 시스템(Material Design, Apple HIG, Nielsen Norman Group 등) 문서가 있는가?

### Q2. KPI 카드 시각화 — 프로그레스 바 + 비율 표시
- 설비 상태 비율(신규/정상/노후)을 KPI 카드 내에 표시할 때 사용하기 좋은 Tailwind CSS 기반 패턴은 무엇인가?
- shadcn/ui 또는 Tremor에서 제공하는 컴포넌트 중 이 용도에 적합한 것이 있는가? (공식 docs 기준)
- 스택형 프로그레스 바(3가지 색상 분할)를 Tailwind만으로 구현할 때 권장 방식은?

### Q3. 간트 차트 가독성 개선
- 현재 간트 차트는 CSS `position: absolute` + `%` 방식으로 직접 구현되어 있음
- 월별 세로 그리드라인을 이 구조에서 추가하는 좋은 방법은?
- Next.js App Router + Tailwind 환경에서 간트 차트를 더 쉽게 구현할 수 있는 경량 라이브러리가 있는가?
  - 평가 기준: Next.js 15 호환성, Server Component 아닌 Client Component에서 동작, 번들 크기 최소화, 한국어 라벨 지원
  - 비교 대상 예시: `frappe-gantt`, `dhtmlx-gantt`, `@syncfusion/ej2-gantt`, react-gantt 계열

### Q4. 대형 테이블 UX — 컬럼 우선순위와 점진적 공개
- 10개 컬럼 테이블에서 "핵심 컬럼 고정 + 부가 컬럼 접기(expandable)" 패턴의 UX 모범 사례는?
- Tailwind + shadcn/ui 환경에서 컬럼 토글 또는 확장 행(expanded row) 구현 시 권장 패턴은?
- 모바일이 아닌 데스크탑 전용 대시보드에서 가로 스크롤 vs 컬럼 접기 중 어느 것이 더 권장되는가?

### Q5. 설비 상태 미니 시각화 — 리스트 행 내 도트/바 표시
- 리스트 행 안에 상태 분포를 표현하는 미니 시각화(도트, 세그먼트 바 등) 패턴이 실제 대시보드 제품에서 어떻게 사용되는가?
- Tailwind로 구현 가능한 간단한 세그먼트 바(3색) 예시 코드나 패턴이 있는가?

## 제약

- **프레임워크**: Next.js 15 App Router 필수. Pages Router 방식 제안 불필요
- **스타일**: Tailwind CSS 기반. CSS-in-JS(styled-components 등) 제안 불필요
- **번들 크기**: 추가 라이브러리는 최소화 선호. 직접 구현 가능하면 라이브러리 불필요
- **MVP 범위**: 인터랙션은 단순하게. 애니메이션 복잡도 높은 방향 불필요
- **데이터**: 로컬 JSON 기반. 실시간 연동 없음
- **언어**: 한국어 UI 전용
